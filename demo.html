This actually runs two real TensorFlow.js workloads — a vision op (a 2-D convolution) and an NLP op (attention-style matmul + softmax) — over and over, under the WASM backend and the plain-JS CPU backend. Each run is one noisy sample. As samples pile up, the histograms sharpen and the running averages converge on each backend's true speed — you can watch the noise wash out.
