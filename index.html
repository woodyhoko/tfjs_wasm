When is WASM
actually faster?

TensorFlow.js runs inference on three interchangeable backends — WebGL, WebAssembly, and plain JS. This project benchmarks WASM vs. CPU across operation shapes and three browsers via Selenium-automated timing, answering: when is WASM worth it, and when does it regress?

CHROME / EDGE

3–5× speedup

WASM SIMD + threads is the fastest CPU path; Edge matches Chrome since both use V8's WASM compilers.

FIREFOX

Version-dependent

SIMD support varies by version; threading needs explicit COOP/COEP server headers.

SMALL TENSORS

WASM regresses

Below ~1 KB, WASM's fixed per-call overhead loses to plain JS — crossover around 10–50 KB.

METHOD

Selenium timing

50 repeats after warm-up, performance.now(), automated across all three browsers.
